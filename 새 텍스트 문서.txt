다음 요구사항을 만족하는 완전한 구현 방법을 만들어줘.

[목표]
- Android Native PDF Viewer를 사용하여 PDF를 벡터 방식으로 렌더링
- 인터넷 없이 (오프라인) 동작
- 로컬 저장소에 있는 PDF 파일을 직접 열기
- 확대/축소(핀치 줌), 스크롤 지원
- 도면(PDF)을 확대해도 깨지지 않아야 함

[기술 조건]
- 기존 앱은 Kivy 기반 (Python)
- pyjnius를 사용해서 Android(Java) 코드와 연결
- Android 라이브러리는 Pdfium 기반 (예: AndroidPdfViewer)
- WebView는 사용하지 않음

[필수로 포함해줘]
1. Android PDF Viewer 라이브러리 사용 방법 (예: AndroidPdfViewer)
2. 필요한 Java 코드 (PDF View 생성 및 파일 로드)
3. Python 코드 (pyjnius로 Java View를 호출하는 코드)
4. Kivy UI 위에 Android View를 올리는 방법
5. PDF 파일 경로를 Android에서 읽을 수 있도록 처리하는 방법
6. buildozer.spec 설정 (매우 중요)
   - android.permissions
   - android.gradle_dependencies
   - 필요한 설정 전부 포함
7. 앱 실행 시 PDF를 여는 전체 흐름

[추가 요구사항]
- Kivy 화면은 유지하고, PDF 영역만 Native View로 표시
- 기존 버튼(UI)은 그대로 사용 가능해야 함
- 슬라이드(다음/이전 PDF 로딩) 확장 가능하도록 구조 설계

[중요]
- 실제로 APK 빌드 가능한 수준으로 작성
- 누락 없이 전체 코드 흐름을 제공
- 오류 발생 가능성이 있는 부분은 주석으로 설명














문제 상황:

* Python 코드에서는 `com.github.barteksc.pdfviewer.PDFView`를 사용하도록 구현 완료
* 하지만 APK 실행 시 PDF가 표시되지 않음 (강제종료됨)
* WebView 방식은 완전히 제거된 상태
* 원인은 buildozer.spec 설정 문제로 판단됨

현재 buildozer.spec의 requirements는 아래와 같음:
requirements = python3,kivy,openpyxl,et_xmlfile,jdcal,pyjnius,android,pysmb,pyasn1,six,tqdm

요구사항:

1. Native PDF Viewer가 정상 동작하도록 buildozer.spec을 수정하라
2. 반드시 필요한 android.gradle_dependencies를 추가하라
3. 불필요한 requirements는 정리하라 (최소 구성으로)
4. 수정된 buildozer.spec 전체를 완성된 형태로 출력하라
5. 왜 해당 설정이 필요한지도 간단히 설명하라

추가 조건:

* PDF는 인터넷 없이 오프라인에서 열려야 함
* WebView 또는 Google Docs Viewer는 절대 사용하지 않음
* 반드시 android-pdf-viewer 라이브러리를 사용해야 함
* Buildozer 기준으로 바로 빌드 가능한 상태여야 함

목표:

buildozer.spec:
android.gradle_dependencies = com.github.barteksc:android-pdf-viewer:2.8.2
코드:
PDFViewClass = autoclass('com.github.barteksc.pdfviewer.PDFView')
PDF 파일 경로 처리 수정
기존:
FileClass(os.path.abspath(path))

👉 아래로 변경:
FileClass(path)

Android 11 이상에서 Scoped Storage 문제로 abs 경로 사용 시 크래시 발생함

기존:
mActivity.addContentView(self.native_pdf_view, lp)

👉 아래로 변경:
decor = mActivity.getWindow().getDecorView()
decor.addView(self.native_pdf_view, lp)

addContentView는 일부 기기에서 크래시 유발
PDF 로드 방식 수정 (체인 호출 금지)
기존:
self.native_pdf_view.fromFile(file_obj).enableSwipe(True).load()

👉 아래처럼 분리:
config = self.native_pdf_view.fromFile(file_obj)
config.enableSwipe(True)
config.load()

if not os.path.exists(path):

현재 프로젝트는 Kivy 기반 Android 앱이며, 기존 PDF 뷰어 구현 방식에서 빌드 실패 및 성능 문제가 지속적으로 발생하고 있다.

특히 Python 기반 PDF 렌더링 또는 Android 네이티브 PDF 라이브러리(Pyjnius, android-pdf-viewer 등)를 사용하는 방식은 다음 문제를 발생시킨다:

APK 빌드 실패 빈번
빌드 시간 과도하게 길어짐
로컬 빌드 시 시스템 불안정 (블루스크린 발생)
유지보수 난이도 매우 높음

따라서 기존 PDF 처리 방식을 전면 폐기하고, 아래 방식으로 구조를 변경한다.

[목표]

오프라인 환경에서 PDF 파일을 안정적으로 표시
빌드 안정성 확보 (GitHub Actions 또는 외부 CI 포함)
Kivy 구조는 유지
확대/스크롤이 자연스럽게 동작해야 함

[필수 구현 방식]

PDF.js를 프로젝트 내부에 포함 (CDN 사용 금지)
Android WebView를 통해 PDF 표시
로컬 파일 경로(file://) 기반으로 PDF 열기

[구현 요구사항]

PDF.js 전체 파일을 프로젝트에 포함
viewer.html 포함
assets 또는 적절한 경로에 배치
Kivy 앱에서 WebView를 호출하여 아래 형식으로 PDF 열기
file:///android_asset/pdfjs/web/viewer.html?file=file:///android_asset/파일명.pdf
buildozer.spec 수정
android.add_assets에 pdfjs 폴더 포함
WebView 사용에 필요한 설정 추가
기존 PDF 관련 라이브러리 제거
PyMuPDF, pdfium, android-pdf-viewer 등 전부 제거
코드 구조는 단일 파일(main.py) 유지
모든 기능 하나의 파일에 통합
앱 내에서 PDF 파일 선택 또는 리스트와 연동 가능하도록 확장 고려

[중요]

인터넷 없이 완전히 동작해야 함
PDF 렌더링은 절대 Python에서 처리하지 말 것

package org.test.myapp;

import android.os.Bundle;
import android.widget.*;
import android.view.*;
import android.graphics.Color;

import com.github.barteksc.pdfviewer.PDFView;

import java.io.File;
import java.util.ArrayList;

import org.kivy.android.PythonActivity;
import org.kivy.android.PythonUtil;

public class PdfActivity extends PythonActivity {

    static ArrayList<String> fileList;
    static int currentIndex = 0;

    public static void open(ArrayList<String> files, int index) {
        fileList = files;
        currentIndex = index;

        PythonActivity.mActivity.runOnUiThread(() -> {
            PythonActivity.mActivity.startActivity(
                new android.content.Intent(PythonActivity.mActivity, PdfActivity.class)
            );
        });
    }

    PDFView pdfView;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);

        LinearLayout root = new LinearLayout(this);
        root.setOrientation(LinearLayout.VERTICAL);

        pdfView = new PDFView(this, null);

        LinearLayout btnLayout = new LinearLayout(this);
        btnLayout.setOrientation(LinearLayout.HORIZONTAL);

        Button btnPrev = new Button(this);
        btnPrev.setText("이전");

        Button btnNext = new Button(this);
        btnNext.setText("다음");

        Button btnDone = new Button(this);
        btnDone.setText("완료");
        btnDone.setBackgroundColor(Color.GREEN);

        Button btnShort = new Button(this);
        btnShort.setText("부족");
        btnShort.setBackgroundColor(Color.YELLOW);

        Button btnRework = new Button(this);
        btnRework.setText("재작업");
        btnRework.setBackgroundColor(Color.RED);

        btnLayout.addView(btnPrev);
        btnLayout.addView(btnNext);
        btnLayout.addView(btnDone);
        btnLayout.addView(btnShort);
        btnLayout.addView(btnRework);

        root.addView(pdfView, new LinearLayout.LayoutParams(
                ViewGroup.LayoutParams.MATCH_PARENT, 0, 1));
        root.addView(btnLayout);

        setContentView(root);

        loadPdf();

        btnPrev.setOnClickListener(v -> {
            if (currentIndex > 0) {
                currentIndex--;
                loadPdf();
            }
        });

        btnNext.setOnClickListener(v -> {
            if (currentIndex < fileList.size() - 1) {
                currentIndex++;
                loadPdf();
            }

    }

    private void loadPdf() {
        try {
            String path = fileList.get(currentIndex);
            File file = new File(path);

            pdfView.fromFile(file)
                    .enableSwipe(true)
                    .swipeHorizontal(true)
                    .enableDoubletap(true)
                    .load();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    private void sendStatus(String status) {
        try {
            String filePath = fileList.get(currentIndex);
            String fileName = new File(filePath).getName();
            String itemCode = fileName.replace(".pdf", "");

            // 🔥 핵심: pyjnius로 Python 함수 호출
            PythonUtil.callPythonFunction(
                "main",                      // main.py
                "update_status_from_java",  // 함수명
                itemCode,
                status
            );

        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

엑셀 저장 안되는 문제 (핵심)

이건 단순 UI 문제가 아니라
Android 저장 권한 + 경로 문제다.

지금 코드 보면:

final file = File(path);
file.writeAsBytesSync(fileBytes, flush: true);

👉 이 방식은 Android 11 이상에서 실패할 확률 매우 높음

🚨 문제 원인 (중요)

너 상황 정확히 말해주면:

👉 /sdcard/... 직접 접근
👉 Android 11+ → Scoped Storage 막힘
👉 WRITE_EXTERNAL_STORAGE 거의 무용지물

그래서 저장은 실패하지만 에러 없이 지나가는 경우 많음




💡 추가 (진짜 중요한 팁)

지금 저장 안되는 이유 90%는 이거다:

👉 엑셀 파일이 다른 앱에서 열려있음

그래서 이것도 반드시 확인:

if (file.existsSync()) {
  try {
    file.deleteSync();
  } catch (e) {
    print("삭제 실패 = 다른 앱 사용중");
    return false;
  }
}

   * 이유:
       1. Tree URI 문제: 폴더 선택기로 가져온 경로가 실제 파일 경로가 아닌 안드로이드 내부의 '가상 경로'일 수 있습니다. 이 경우 뷰어 엔진이 실제 파일을 찾지 못합니다.
       2. 읽기 권한 누락: 엑셀 저장과 마찬가지로, 특정 폴더 안의 개별 파일들에 대한 '읽기 스트림' 권한이 뷰어 엔진에 전달되지 않았을 가능성이 큽니다.
> PDFX와 무엇이 다른거야? PDFX는 뷰어기능 자체는 정상이었고, 다만, 축소가 내가 원하는 대로 되지 않았었어.

   * 장점: 시스템이 제공하는 공식 통로를 쓰기 때문에 100% 저장이 보장됩니다.
   * 변경점: 파일을 선택할 때 시스템 팝업이 뜨고, 거기서 한 번 더 "이 파일에 대한 쓰기 권한을 허용하시겠습니까?"라는 물음에 승인하는 과정이 들어갑니다.

  대안 2: 메모리 기반 강제 로딩 (PDF 문제 해결)

   * 장점: 경로가 가짜든 진짜든 상관없이 데이터를 읽을 수만 있다면 도면이 무조건 뜹니다.

  대안 3: 작업 폴더의 이동
   * 방법: 공용 폴더(Download)가 아닌, 앱 전용 내부 저장소에 파일을 복사해 놓고 작업합니다.
   * 장점: 보안 정책의 영향을 받지 않아 저장과 읽기가 매우 자유롭습니다.
   * 단점: 작업이 끝난 뒤 엑셀 파일을 다시 밖으로 내보내기(Export) 해야 하는 번거로움이 있습니다.

✅ 어떤 방식으로 가능하냐

Flutter에서 SMB는 보통 라이브러리로 처리한다.

대표적으로:

👉 smb_connect

🔥 가능한 기능

SMB로 할 수 있는 것들:

✔ 공유폴더 목록 조회
✔ 파일 리스트 가져오기
✔ PDF / Excel 다운로드
✔ 로컬 저장 후 열기

⚠️ 중요한 구조 (이게 핵심이다)

Flutter에서는 SMB 파일을

👉 “직접 열지 않는다”

👉 반드시 이렇게 해야 한다:

SMB → 로컬 다운로드 → 열기
📌 네가 원하는 구조 (완벽히 가능)

너가 말한 요구사항:

SMB에서 선택하면
👉 로컬에 복사
👉 PDF / Excel 열기

👉 이건 Flutter에서 정석 구조다

💡 전체 흐름
[SMB 서버]
   ↓
파일 선택
   ↓
앱이 다운로드
   ↓
/sdcard/Download/CheckSheet/ 저장
   ↓
PDF / Excel 열기
🧠 왜 이렇게 해야 하냐

👉 Android 보안 때문

SMB 경로 → 직접 접근 ❌
로컬 파일 → 접근 가능 ✔
⚠️ 주의사항 (중요)
1️⃣ 권한
Permission.manageExternalStorage

또는 SAF

2️⃣ 속도
PDF 크면 다운로드 시간 있음

👉 해결:

진행바 표시 추천

3️⃣ 파일 덮어쓰기
if (file.existsSync()) file.deleteSync();
