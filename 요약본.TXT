[ 체크시트(CheckSheet) 앱 개발 기술 명세서 ]

이 문서는 앱을 처음부터 다시 개발하거나 기능을 복구할 때, 현재의 완성된 상태에 도달하기 위한 핵심 로직과 주의사항을 정리한 것입니다.

1. 프로젝트 개요 및 기술 스택
   - 플랫폼: Flutter (Dart) 기반 안드로이드 앱
   - 패키지명: org.example.checksheet
   - 앱 이름: 체크시트 (한글 지정)
   - 핵심 라이브러리:
     * excel: ^4.0.6 (데이터 로드/저장)
     * pdfrx: 1.3.5 (PDF 렌더링 엔진 - 버전 고정 권장)
     * shared_preferences: 설정값(IP, ID, PW, 공정목록) 저장
     * permission_handler: 외부 저장소 접근 권한 관리

2. 데이터 구조 및 엑셀 연동 (ExcelService)
   - 헤더 구성: [no, 품목코드, 수량, 완료, 보완, 공정, 비고] (7개 고정 컬럼)
   - 부분 제목(Subheading) 로직: 
     * 엑셀에서 'no' 컬럼이 비어있고 '품목코드'만 있는 행을 '부분 제목'으로 인식.
     * 리스트 출력 시 배경색을 다르게 하고, 필터링 시 해당 섹션의 그룹장 역할을 함.
   - 행 삭제 시 주의점: 부분 제목을 삭제할 때 반드시 해당 행의 인덱스(realIndex)를 포함해야 엑셀 저장 시 누락되지 않음.

3. 리스트 화면 핵심 구현 (ChecklistScreen)
   - 한글 입력 해결 (중요): 
     * 비고란(Remarks) 입력 시 글자 깨짐(받침 누락)을 방지하기 위해 별도의 StatefulWidget(_RemarksCell)로 분리 필수.
     * 입력을 마칠 때까지 TextEditingController가 초기화되지 않도록 격리해야 함.
   - 포커스 제어: 
     * 다른 곳 터치 시 즉시 키패드를 닫기 위해 _forgetFocus()와 FocusNode 관리 필요.
   - 정렬 및 필터링: 
     * 모든 헤더(완료, 보완, 공정 포함) 클릭 시 오름차순/내림차순 정렬 지원.
     * 상단 요약 바 클릭 시 '미완료 항목만 보기' 필터링 연동.

4. PDF 뷰어 및 네비게이션 (PdfViewerScreen)
   - 한 손 조작 UI:
     * 화면 상/하단 끝 여백에 [<- 이전], [다음 ->] 화살표 배치.
     * 아이콘 크기는 24, 색상은 리스트의 품목코드와 동일한 파란색 적용.
     * 본문을 가리지 않도록 Positioned(top: 80, bottom: 80) 등으로 위치 최적화.
   - 상태 버튼 디자인: 
     * [완료, 보완, 공정] 버튼 구성. 
     * 내용이 없을 때는 라벨을 크게(15), 내용이 입력되면 라벨은 작게(12) 하고 실제 내용(부족, 레이저 등)을 크게(16, Bold) 표시하여 가독성 증대.

5. 네트워크 및 동기화 (SmbService & Native)
   - SMB 통신: 
     * 안드로이드 전용 로직이 필요하여 Kotlin(SmbHandler.kt)을 활용한 Native Channel 연결.
   - 접속 테스트: 성공 시 해당 서버의 모든 공유 폴더 목록(List)을 텍스트로 사용자에게 보여줌.
   - 파일 경로: 모든 다운로드 파일은 `/storage/emulated/0/Download/CheckSheet/` 경로에 저장 및 관리.

6. 기타 주의사항 및 설정
   - 자동 저장: 설정 ON일 경우 체크 상태 변경, 비고 입력, 행 삭제 즉시 엑셀 저장 로직 호출.
   - 공정 관리: 사용자가 직접 공정 목록을 추가/삭제/순서변경(ReorderableListView) 할 수 있도록 설정창 제공.
   - 권한: AndroidManifest.xml에 MANAGE_EXTERNAL_STORAGE 권한 명시 필수.
