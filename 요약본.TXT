[ 체크시트(CheckSheet) 앱 재구축 및 개발 지침서 ]

본 문서는 앱을 처음부터 다시 제작하거나 주요 기능을 복구할 때, 시행착오 없이 현재의 완성된 결과물에 도달하기 위한 핵심 기술 명세와 로직을 정리한 가이드입니다.

1. 기술 스택 및 환경 설정
   - 언어 및 프레임워크: Flutter (Dart)
   - 타겟 플랫폼: Android (권한: MANAGE_EXTERNAL_STORAGE 필수)
   - 핵심 라이브러리 (버전 준수 필수):
     * excel: ^4.0.6 (엑셀 읽기/쓰기)
     * pdfrx: 1.3.5 (PDF 렌더링 - ❗excel과의 호환성을 위해 반드시 이 버전 고정)
     * flutter_launcher_icons: ^0.13.1 (앱 아이콘 자동 생성 및 관리)
     * jcifs-ng: (안드로이드 네이티브 SMB 통신용 - Kotlin에서 구현)
     * shared_preferences, permission_handler, path, file_picker

2. 엑셀 데이터 로드 및 관리 로직
   - "null" 문자열 방어 (이중 보안): 엑셀 빈 셀을 읽을 때 "null" 글자가 들어오는 버그 대응을 위해 `ExcelService`와 `ItemModel` 생성자 양쪽에서 빈 값("") 치환 로직 구현 필수.
   - 가상 번호 부여 (Display Numbering): 
     * 'no'가 있는 행은 메인 번호로 인식.
     * 'no'가 비어있고 '수량'이 있는 행은 "메인번호-순번" 형태(예: 2-1, 2-2)로 가상 번호 자동 생성.
   - 부분제목(Subheading) 처리: 'no'와 '수량'이 모두 없고 '품목코드'만 있는 행을 섹션 구분선으로 인식. 해당 타이틀을 하위 품목들의 `subheadingTitle` 필드에 자동으로 전파하여 저장.

3. 리스트 화면(ChecklistScreen) 및 UI 최적화
   - 필터 리셋 버튼 배치: 정렬/필터 적용 시 나타나는 '필터리셋' 버튼은 AppBar actions의 가장 앞(왼쪽)에 배치하여, 기존 '새로고침/닫기/자동저장' 버튼들의 위치가 변하지 않도록 설계.
   - 한글 입력 최적화: 비고(Remarks) 입력 시 한글 받침 끊김 현상을 방지하기 위해 별도의 StatefulWidget(`_RemarksCell`)로 분리 구현.
   - 포커스 해제: 입력 완료(onSubmitted) 또는 화면 어디든 터치 시 `_forgetFocus()`를 통해 키패드를 즉시 닫고 가상 포커스 노드(`_dummyFocusNode`)로 이동.
   - 정렬 로직: 모든 헤더 클릭 시 정렬 지원. 가상 번호(2-10 등)는 '-' 기준 숫자 비교(`_compareDisplayNo`) 로직 적용.
   - 품목코드 3단계 순환 정렬: 헤더 클릭 시 '오름차순 → 내림차순 → 정렬 해제(원래 순서)' 순으로 반복 순환하여 사용자 편의성 극대화.

4. 고급 필터 및 검색 시스템
   - 'No' 헤더 3단계 순환 필터:
     * 1단계: 전체 표시 (기본)
     * 2단계: 엑셀 원본 NO가 있는 항목만 표시 (헤더 노란색)
     * 3단계: '하위가 없는 원본' + '하위 항목들'만 표시 (헤더 텍스트: -No, 노란색)
   - 부분제목 다중 선택 필터링 (신규):
     * 부분제목 보기 모드에서 왼쪽 체크박스를 통해 여러 섹션을 동시에 선택 가능.
     * 선택 시 하단에 [선택 항목 보기], [모두 해제], [취소] 버튼이 포함된 '확인 바'가 나타나며 필터링 제어.
     * 텍스트 클릭 시에는 기존처럼 단일 섹션 즉시 필터링 유지.
   - 지능형 필터 항목 표시 (Context-Aware Filter):
     * 다른 필터 상태를 반영하여 현재 리스트에 존재하는 데이터 기반으로만 옵션 구성.
     * 공정(Process) & 수량(Quantity): 현재 결과에 없는 항목은 목록에서 자동 제외.
     * 완료(Complete) & 보완(Complement): 현재 결과에 없는 항목은 목록에 유지하되, 30% 투명도로 비활성화 처리 및 선택 방지.
   - 필터 편의 기능: 모든 체크박스 필터(공정, 수량 등) 상단에 **[전체 선택] [전체 해제]** 버튼 제공.
   - 검색창 고도화: 메인 및 PDF 뷰어 검색창에 입력 내용 즉시 삭제용 **'X' 버튼** 탑재. 검색 중에도 '필터리셋' 연동 지원.
   - 임시 노출(Temporary Visibility): 뷰어 검색으로 숨겨진 항목 선택 시 리스트에 강제 노출 및 하이라이트. 터치 시 복구.

5. 행 삭제(편집) 및 순서 변경 모드 (UX 고도화)
   - 부분제목 순서 변경 모드 (신규):
     * 부분제목 보기 모드에서 우측 'reorder' 아이콘 클릭 시 진입. 진입 시 현재 상태 백업(`_preReorderItems`)을 통해 **[취소]** 기능 지원.
     * ReorderableListView를 사용하여 섹션 위치를 변경하면, 해당 섹션에 포함된 **모든 하위 데이터가 일괄적으로 함께 이동**하여 데이터 무결성 유지. (리스트 끝으로 이동 시 발생하는 인덱스 버그 수정 완료)
     * 순서 변경 모드에서는 검색 및 헤더를 숨겨 조작 실수 방지.
   - 행 삭제(편집) 모드 UI 정문화: 편집 모드 진입 시 부분제목 행에서 필터용 체크박스와 순서 변경 버튼을 자동으로 숨겨 오조작 방지.
   - 시각적 연쇄 선택 (삭제 모드): 부분제목 선택 시 해당 섹션의 모든 하위 항목 체크박스가 시각적으로 즉시 파랗게 선택됨.
   - UI 레이아웃: 일반 항목은 체크박스가 왼쪽, 부분제목은 우측 끝에 배치하여 시각적 위계 구분.
   - 삭제 안내: "선택한 섹션 및 하위 항목 포함 총 X개를 삭제하시겠습니까?"와 같이 실제 삭제 수량을 정확히 안내.
   - 로컬 PDF 연동 삭제: 로컬 폴더 모드일 때만 삭제 창에 옵션 제공. 체크 시 리스트 행과 로컬 PDF 파일을 동시에 영구 삭제.

6. PDF 뷰어 화면(PdfViewerScreen)
   - 제목 UI: AppBar 2단 구성 (부분제목 / 품목코드).
   - 이원화된 내비게이션: 
     * 검색: 전체 데이터를 대상으로 수행.
     * 이전/다음 이동: 현재 메인 리스트의 필터링된 순서를 엄격히 준수.
   - 네비게이션 배치: 좌측 하단(`bottom: 80, left: 5`) 플로팅 화살표 버튼.
   - 상태 동기화: 완료, 보완, 공정(커스텀 색상 반영), 비고 상태 즉시 메인과 연동.

7. 공정 관리 및 색상 커스터마이징
   - 기본 공정 목록: '레이저...인쇄, 버핑, 용접, 보류, 사급, 완료' (버핑은 용접 위 배치).
   - Accent Bar 스타일: 공정 칸 왼쪽 끝에 4px 진한 색상 바를 표시하고 배경은 연한 색상을 채워 가독성과 구분을 동시 확보.
   - 기본 색상 사양 (사용자 미지정 시):
     * 완료: 보라색 (Colors.purple)
     * 보류: 빨간색 (Colors.red)
     * 용접, 도장, 도금, 인쇄: 주황색 (Colors.orange)
     * 기타 공정: 회색 계열 (Colors.blueGrey)
   - 색상 선택: 설정에서 15가지 표준 팔레트를 통해 공정별 고유 색상 지정 가능.
   - UI 밀도: 설정 내 공정 리스트의 행 간격을 절반으로 축소하여 관리 편의성 증대.

8. SMB 공유 폴더 연동 (Native Bridge)
   - Dart(`SmbService`) <-> Kotlin(`SmbHandler`) 간 MethodChannel 통신.
   - 매 호출 시 인증 정보 전달 및 대소문자 무관 파일 탐색 로직 적용.
