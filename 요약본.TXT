[ 체크시트(CheckSheet) 앱 개발 기술 명세서 - 최종본 ]

이 문서는 앱을 처음부터 다시 개발하거나 기능을 복구할 때, 현재의 완성된 상태에 도달하기 위한 핵심 로직과 주의사항을 정리한 것입니다.

1. 프로젝트 개요 및 기술 스택
   - 플랫폼: Flutter (Dart) 기반 안드로이드 앱
   - 패키지명: org.example.checksheet / 앱 이름: 체크시트
   - 핵심 라이브러리:
     * excel: ^4.0.6 (데이터 관리)
     * pdfrx: 1.3.5 (PDF 엔진 - 버전 고정 필수)
     * shared_preferences / permission_handler

2. 가상 번호 시스템 (Display Numbering)
   - 구현 목적: 엑셀의 'no' 컬럼이 비어있어도 수량이 있는 하위 품목을 식별하기 위함.
   - 번호 부여 알고리즘:
     * 메인 번호(숫자 있음): 해당 번호를 메인으로 저장하고 하위 인덱스 초기화.
     * 하위 품목(no 없음, qty 있음): "메인번호-순번" 형태로 가상 번호 생성 (예: 2-1, 2-2).
   - 정렬 로직: 단순 문자열 정렬이 아닌, '-'를 기준으로 쪼개서 숫자 크기를 비교하는 `_compareDisplayNo` 함수 필수 구현.

3. 데이터 정화 및 "null" 방어 (Double-Layer Defense)
   - 원인: 엑셀 라이브러리가 빈 셀을 읽을 때 문자열 "null"을 반환하는 버그 대응.
   - 1차 방어 (ExcelService): `_getSafe` 함수에서 모든 "null" 문자열을 즉시 빈 값("")으로 치환.
   - 2차 방어 (ItemModel): 생성자(Constructor) 내부에서 모든 필드의 "null" 여부를 전수 조사하여 필터링.
   - 저장 로직: 깨끗해진 데이터를 사용하여 엑셀 저장 시 "null" 글자가 적히는 현상 원천 차단.

4. 리스트 화면 핵심 구현 (ChecklistScreen)
   - 한글 입력 해결: 비고란(Remarks)은 별도의 StatefulWidget(_RemarksCell)로 분리하여 한글 받침 끊김 방지.
   - 포커스 제어: 입력을 마치거나 밖을 터치하면 `_forgetFocus()`를 통해 키패드를 즉시 닫음.
   - 전체 컬럼 정렬: [No, 품목코드, 수량, 완료, 보완, 공정, 비고] 모든 헤더에 정렬 기능 연동.

5. PDF 뷰어 및 네비게이션 (PdfViewerScreen)
   - 한 손 조작 UI: [<- 이전] [다음 ->] 화살표 버튼을 좌측 하단 여백(`bottom: 80, left: 5`)에 나란히 배치.
   - 아이콘 사양: `Icons.arrow_back/forward`, 크기 `24`, 색상은 품목코드와 동일한 파란색 적용.
   - 상태 버튼 디자인: 
     * 내용이 없을 때: 라벨을 크게(15, Bold) 표시.
     * 내용이 있을 때: 라벨은 작게(12), 실제 내용(부족, 용접 등)은 크게(16, Bold) 표시.

6. 네트워크 및 설정
   - SMB 접속: 접속 성공 시 서버 내 공유 폴더 목록을 알림창에 텍스트로 즉시 출력.
   - 공정 관리: 기본 목록에 [레이저, CS, 탭, 버링탭, 헤밍, ZB, 절곡, 압입, 리베팅, 용접, 도장, 도금, 인쇄] 포함.
   - 자동 저장: 설정 ON 시 체크, 비고, 행 삭제 발생 즉시 엑셀 파일 업데이트.

7. 기타 주의사항
   - 권한: AndroidManifest.xml에 MANAGE_EXTERNAL_STORAGE 권한 필수.
   - 경로: 모든 작업 파일은 `/storage/emulated/0/Download/CheckSheet/`를 기준으로 관리.
