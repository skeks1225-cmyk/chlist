[ 체크시트(CheckSheet) 앱 재구축 및 개발 지침서 ]

본 문서는 앱을 처음부터 다시 제작하거나 주요 기능을 복구할 때, 시행착오 없이 현재의 완성된 결과물에 도달하기 위한 핵심 기술 명세와 로직을 정리한 가이드입니다.

1. 기술 스택 및 환경 설정
   - 언어 및 프레임워크: Flutter (Dart)
   - 타겟 플랫폼: Android (권한: MANAGE_EXTERNAL_STORAGE 필수)
   - 핵심 라이브러리 (버전 준수 필수):
     * excel: ^4.0.6 (엑셀 읽기/쓰기)
     * pdfrx: 1.3.5 (PDF 렌더링 - ❗excel과의 호환성을 위해 반드시 이 버전 고정)
     * flutter_launcher_icons: ^0.13.1 (앱 아이콘 자동 생성 및 관리)
     * jcifs-ng: (안드로이드 네이티브 SMB 통신용 - Kotlin에서 구현)
     * shared_preferences, permission_handler, path, file_picker

2. 엑셀 데이터 로드 및 관리 로직
   - "null" 문자열 방어 (이중 보안): 엑셀 빈 셀을 읽을 때 "null" 글자가 들어오는 버그 대응을 위해 `ExcelService`와 `ItemModel` 생성자 양쪽에서 빈 값("") 치환 로직 구현 필수.
   - 가상 번호 부여 (Display Numbering): 
     * 'no'가 있는 행은 메인 번호로 인식.
     * 'no'가 비어있고 '수량'이 있는 행은 "메인번호-순번" 형태(예: 2-1, 2-2)로 가상 번호 자동 생성.
   - 부분제목(Subheading) 처리: 'no'와 '수량'이 모두 없고 '품목코드'만 있는 행을 섹션 구분선으로 인식. 해당 타이틀을 하위 품목들의 `subheadingTitle` 필드에 자동으로 전파하여 저장.

3. 리스트 화면(ChecklistScreen) 및 UI 최적화
   - 필터 리셋 버튼 배치: 정렬/필터 적용 시 나타나는 '필터리셋' 버튼은 AppBar actions의 가장 앞(왼쪽)에 배치하여, 기존 '새로고침/닫기/자동저장' 버튼들의 위치가 변하지 않도록 설계.
   - 한글 입력 최적화: 비고(Remarks) 입력 시 한글 받침 끊김 현상을 방지하기 위해 별도의 StatefulWidget(`_RemarksCell`)로 분리 구현.
   - 포커스 해제: 입력 완료(onSubmitted) 또는 화면 어디든 터치 시 `_forgetFocus()`를 통해 키패드를 즉시 닫고 가상 포커스 노드(`_dummyFocusNode`)로 이동.
   - 정렬 로직: 모든 헤더 클릭 시 정렬 지원. 가상 번호(2-10 등)는 '-' 기준 숫자 비교(`_compareDisplayNo`) 로직 적용.
   - 품목코드 3단계 순환 정렬: 헤더 클릭 시 '오름차순 → 내림차순 → 정렬 해제(원래 순서)' 순으로 반복 순환하여 사용자 편의성 극대화.
   - **부분제목 UI 고도화 (개선)**:
     * 행 높이를 **80px로 고정**하여 정보 밀도가 높은 3단 레이아웃 적용.
     * **논리적 3단 분리**: 3번째 언더바(`_`)를 기준으로 업체/일정 정보(1행)와 모델명/상세 사양(2행)을 자동 분리 표시.
     * **스타일 및 정렬 통일**: 1행과 2행의 글꼴(Bold), 크기(14), 색상을 동일하게 적용하여 일관성 부여. 2행은 1행의 제목 시작점과 일치하도록 들여쓰기 정렬.
     * **텍스트 자동 맞춤 (Fit)**: 제목이 가로 폭을 넘칠 경우 `FittedBox`를 통해 글자 크기를 자동으로 조절하여 정보 누락 방지.
     * **실시간 요약**: 3행에 총 아이템 수, 완료 수 및 진행률(%) 상시 표시.
     * **전체 완료 강조**: 섹션 내 모든 항목 완료 시 배경색을 연한 초록색으로 변경하고 **🏆(트로피)** 아이콘 표시.
     * 일관성 유지: 메인 UI와 리셋 섹션 선택 다이얼로그 양쪽에 동일한 3단 레이아웃 로직 적용.

4. 고급 필터 및 검색 시스템
   - 'No' 헤더 3단계 순환 필터:
     * 1단계: 전체 표시 (기본)
     * 2단계: 엑셀 원본 NO가 있는 항목만 표시 (헤더 노란색)
     * 3단계: '하위가 없는 원본' + '하위 항목들'만 표시 (헤더 텍스트: -No, 노란색)
   - 부분제목 다중 선택 필터링 (신규):
     * 부분제목 보기 모드에서 왼쪽 체크박스를 통해 여러 섹션을 동시에 선택 가능.
     * 선택 시 하단에 [선택 항목 보기], [모두보기], [모두 해제], [취소] 버튼이 포함된 '확인 바'가 나타나며 필터링 제어.
     * **[모두보기]**: 현재 선택을 초기화하고 즉시 전체 리스트 상태로 복구하는 기능.
     * 텍스트 클릭 시에는 기존처럼 단일 섹션 즉시 필터링 유지.
   - 공정 필터 정렬 규칙 (신규):
     * 필터 다이얼로그 내 항목 순서는 **(빈칸) → 설정된 관리 순서 → 미등록 공정 → 완료** 순으로 고정하여 일관성 유지.
   - 필터 UI 디자인 통일 (Accent Bar): 공정 필터 다이얼로그의 각 항목 앞에 메인 UI와 동일한 세로 색상바를 추가하여 가독성과 디자인 일관성 확보.
   - 지능형 필터 항목 표시 (Context-Aware Filter):
     * 다른 필터 상태를 반영하여 현재 리스트에 존재하는 데이터 기반으로만 옵션 구성.
     * 공정(Process) & 수량(Quantity): 현재 결과에 없는 항목은 목록에서 자동 제외.
     * 완료(Complete) & 보완(Complement): 현재 결과에 없는 항목은 목록에 유지하되, 30% 투명도로 비활성화 처리 및 선택 방지.
   - 필터 편의 기능: 모든 체크박스 필터(공정, 수량 등) 상단에 **[전체 선택] [전체 해제]** 버튼 제공.
   - 고급 검색창 삭제 버튼: 메인 및 PDF 뷰어 검색창에 입력 내용 즉시 삭제용 'X' 버튼 탑재.
   - 빌드 결과물 이름 (신규): GitHub Actions를 통해 생성되는 설치 파일 이름을 **checksheet.apk**로 고정하여 다운로드 안정성 및 호환성 확보.
   - 임시 노출(Temporary Visibility): 뷰어 검색으로 숨겨진 항목 선택 시 리스트에 강제 노출 및 하이라이트. 터치 시 복구.
   - **통합 리셋 시스템 (개선)**:
     * 상단 '리셋' 버튼을 통한 일원화된 진입점 제공 (섹션별 개별 버튼 제거로 UI 간소화).
     * **단계별 프로세스**: 범위 선택(전체/부분) → 부분 선택(섹션 목록) → 대상 선택(체크상태/비고) 순으로 진행.
     * **선택적 초기화**: '체크상태(완료/보완/공정)'와 '비고란'을 개별적으로 선택하여 리셋 가능. 중요 메모 보존성 향상.
     * **섹션 선택 UI 고도화**:
       - 긴 제목 최적화: 카드 레이아웃 및 줄바꿈(`softWrap`) 적용으로 긴 제목도 잘림 없이 표시.
       - 상세 정보 요약: 각 섹션별 **순번[01]**, **총 아이템 수**, **완료 수 및 진행률(%)**을 실시간 계산하여 표시.
       - 시각적 강조: 전체 완료된 섹션은 **연한 초록색 배경**과 **🏆(트로피)** 아이콘으로 자동 강조.
       - **지능형 상시 슬라이드바**: 항목이 많아 스크롤이 필요한 경우 우측에 두껍고 명확한 슬라이드바가 항상 표시되어 현재 위치 파악 용이. 항목이 적을 땐 자동 숨김.
     * **위치 기반 정밀 로직**: 부분 리셋 시 객체의 Index를 기준으로 하위 아이템을 정확히 식별하여 초기화.
     * 자동 저장 및 필터 재적용과 완벽 연동되어 데이터 무결성 보장.

5. 행 삭제(편집) 및 순서 변경 모드 (UX 고도화)
   - 부분제목 순서 변경 모드 (신규):
     * 부분제목 보기 모드에서 우측 'reorder' 아이콘 클릭 시 진입. 진입 시 현재 상태 백업(`_preReorderItems`)을 통해 **[취소]** 기능 지원.
     * ReorderableListView를 사용하여 섹션 위치를 변경하면, 해당 섹션에 포함된 **모든 하위 데이터가 일괄적으로 함께 이동**하여 데이터 무결성 유지. (리스트 끝으로 이동 시 발생하는 인덱스 버그 수정 완료)
     * 순서 변경 모드에서는 검색 및 헤더를 숨겨 조작 실수 방지.
   - 공정 선택 UI 3열 확장 (신규): 공정 항목 증가에 대응하여 메인 및 PDF 뷰어의 공정 선택 다이얼로그를 **3열 레이아웃**으로 확장.
   - 행 삭제(편집) 모드 UI 정문화: 편집 모드 진입 시 부분제목 행에서 필터용 체크박스와 순서 변경 버튼을 자동으로 숨겨 오조작 방지.
   - 시각적 연쇄 선택 (삭제 모드): 부분제목 선택 시 해당 섹션의 모든 하위 항목 체크박스가 시각적으로 즉시 파랗게 선택됨.
   - UI 레이아웃: 일반 항목은 체크박스가 왼쪽, 부분제목은 우측 끝에 배치하여 시각적 위계 구분.
   - 삭제 안내: "선택한 섹션 및 하위 항목 포함 총 X개를 삭제하시겠습니까?"와 같이 실제 삭제 수량을 정확히 안내.
   - 로컬 PDF 연동 삭제: 로컬 폴더 모드일 때만 삭제 창에 옵션 제공. 체크 시 리스트 행과 로컬 PDF 파일을 동시에 영구 삭제.
   - **행삭제 모드 클릭 동작 최적화 (개선)**:
     * **품목코드 영역**: 행삭제 모드 여부와 관계없이 클릭 시 PDF 뷰어로 항상 이동하여 작업 연속성 보장.
     * **그 외 모든 영역 (No, 완료, 보완, 공정, 비고 등)**: 행삭제 모드일 때는 기존 수정 기능을 차단하고 **행 선택(삭제용 체크)** 기능으로 전환하여 오조작 방지 및 빠른 다중 선택 지원.
     * **비고란 키보드 차단**: 행삭제 모드에서 비고란 클릭 시 키보드가 올라오지 않도록 `IgnorePointer`를 적용하여 쾌적한 행 선택 환경 제공.

6. PDF 뷰어 화면(PdfViewerScreen)
   - 제목 UI: AppBar 2단 구성 (부분제목 / 품목코드).
   - 이원화된 내비게이션: 
     * 검색: 전체 데이터를 대상으로 수행.
     * 이전/다음 이동: 현재 메인 리스트의 필터링된 순서를 엄격히 준수.
   - 네비게이션 배치: 좌측 하단(`bottom: 80, left: 5`) 플로팅 화살표 버튼.
   - 상태 동기화: 완료, 보완, 공정(커스텀 색상 반영), 비고 상태 즉시 메인과 연동.
   - **상태 버튼 색상 정책**: 
     * 완료 버튼: 공정 색상과 관계없이 항상 **녹색(Colors.green)**으로 표시하여 가시성 확보.
     * 공정 버튼: 설정에서 지정한 **커스텀 색상**을 우선 적용 (지정 없을 시 보라/빨강/주황/회색 순).

7. 공정 관리 및 색상 커스터마이징
   - 기본 공정 목록: '레이저...인쇄, 버핑, 용접, 보류, 사급, 완료' (버핑은 용접 위 배치).
   - Accent Bar 스타일: 공정 칸 왼쪽 끝에 4px 진한 색상 바를 표시하고 배경은 연한 색상을 채워 가독성과 구분을 동시 확보.
   - 기본 색상 사양 (사용자 미지정 시):
     * 완료: 보라색 (Colors.purple)
     * 보류: 빨간색 (Colors.red)
     * 용접, 도장, 도금, 인쇄: 주황색 (Colors.orange)
     * 기타 공정: 회색 계열 (Colors.blueGrey)
   - **'완료' 항목 보호 및 고정 로직** (신규):
     * '완료'는 항상 리스트 최하단에 위치하며 드래그를 통한 순서 변경 불가.
     * 신규 공정 추가 시 '완료' 바로 위 칸에 자동 삽입.
     * '완료' 항목은 삭제가 불가능하도록 버튼 비활성화.
   - 색상 선택: 설정에서 15가지 표준 팔레트를 통해 공정별 고유 색상 지정 가능.
   - UI 밀도: 설정 내 공정 리스트의 행 간격을 절반으로 축소하여 관리 편의성 증대.

8. SMB 공유 폴더 연동 (Native Bridge)
   - Dart(`SmbService`) <-> Kotlin(`SmbHandler`) 간 MethodChannel 통신.
   - 매 호출 시 인증 정보 전달 및 대소문자 무관 파일 탐색 로직 적용.

9. 파일 시스템 경로
   - 기본 작업 경로: `/storage/emulated/0/Download/CheckSheet/`
   - 로컬 저장, SMB 다운로드 및 삭제 작업의 기준 폴더.

10. QR 코드 스캔 및 검색 시스템 (신규)
   - 모바일 스캐너 통합: `mobile_scanner` 라이브러리를 활용하여 카메라 기반 QR 코드 인식 구현.
   - 팝업형 스캔 UI: 작업 흐름을 방해하지 않도록 다이얼로그(`QrScannerDialog`) 형태로 스캔 화면 제공.
   - 플래시 및 카메라 전환: 어두운 현장 대응을 위한 플래시 제어 및 전/후면 카메라 전환 기능 포함.
   - **강력한 데이터 정제 로직 (개선)**:
     * **특수 문자 제거**: 실제 현장 데이터인 `<NUL>`, `<NULL>` 텍스트를 자동으로 삭제.
     * **제어 문자 청소**: 줄바꿈(`\n`, `\r`) 및 보이지 않는 유령 문자(ASCII 0-31 범위)를 정규표현식으로 완벽히 제거하여 인식 오류 방지.
     * **접미사 지능형 처리**: 품번 끝에 붙는 `-S` 또는 `-s` 접미사를 대소문자 구분 없이 자동 감지하여 제거 후 본체 품번만 추출.
     * **실시간 피드백**: 스캔 직후 **"스캔: [원본] → 정제: [결과]"** 형태의 스낵바를 띄워 인식 상태를 사용자에게 즉시 알림.
   - **UI 레이아웃 최적화 (개선)**:
     * **공간 확보 (2:1)**: 검색 바와 요약 정보의 비율을 **2:1**로 조정하여 긴 품번도 한눈에 확인 가능하도록 개선.
     * **가변 아이콘**: 검색창에 포커스가 가거나 글자가 입력된 상태에서는 앞의 돋보기 아이콘을 자동으로 숨겨 입력 공간 극대화.
   - 지능형 자동 이동 로직:
     * 메인 리스트: 정제된 검색어로 자동 필터링 및 해당 항목으로 **즉시 자동 스크롤 및 파란색 테두리 하이라이트** 처리.
     * PDF 뷰어: 정제된 검색 결과와 일치하는 항목이 있는 경우 해당 PDF를 **즉시 로드하여 표시**.
   - UI 통합: 검색 바 우측 suffixIcon 영역에 QR 아이콘 버튼을 배치하여 검색 접근성 극대화.
