[ 체크시트(CheckSheet) 앱 재구축 및 개발 지침서 ]

본 문서는 앱을 처음부터 다시 시작하거나 주요 기능을 복구할 때, 시행착오 없이 현재의 완성된 결과물에 도달하기 위한 핵심 기술 명세와 로직을 정리한 가이드입니다.

1. 기술 스택 및 환경 설정
   - 언어 및 프레임워크: Flutter (Dart)
   - 타겟 플랫폼: Android (권한: MANAGE_EXTERNAL_STORAGE 필수)
   - 핵심 라이브러리(버전 준수 필수):
     * excel: ^4.0.6 (엑셀 읽기/쓰기)
     * pdfrx: 1.3.5 (PDF 렌더링 - 엑셀과의 호환성을 위해 반드시 이 버전 고정)
     * flutter_launcher_icons: ^0.13.1 (앱 아이콘 자동 생성 및 관리)
     * jcifs-ng: (안드로이드 네이티브 SMB 통신용 - Kotlin에서 구현)
     * shared_preferences, permission_handler, path, file_picker

2. 엑셀 데이터 로드 및 관리 로직
   - "null" 문자열 방어 (이중 보안): 엑셀 빈 칸을 읽을 때 "null" 글자가 들어오는 버그 방지를 위해 `ExcelService`와 `ItemModel` 생성자 양쪽에서 빈 값("") 치환 로직 구현 필수.
   - 가상 번호 부여 (Display Numbering):
     * 'no'가 있는 행은 메인 번호로 인식.
     * 'no'가 비어있고 '수량'이 있는 행은 "메인번호-순번" 형태(예: 2-1, 2-2)로 가상 번호 자동 생성.
   - 부분제목(Subheading) 처리: 'no'와 '수량'이 모두 없고 '품목코드'만 있는 행을 섹션 구분선으로 인식. 해당 타이틀은 하위 항목들의 `subheadingTitle` 필드에 자동으로 전파되어 저장.
   - 데이터 저장 구조: 시간 데이터는 [완료시간, 공정시간, 보완시간] 순서로 저장되며, 엑셀 헤더에 명시적으로 표시됨.

3. 리스트 화면(ChecklistScreen) 및 UI 최적화
   - 필터 리셋 버튼 배치: 정렬/필터 적용 시 나타나는 '필터리셋' 버튼은 AppBar actions의 가장 왼쪽(첫 번째)에 배치하여, 기존 '새로고침/닫기/자동저장' 버튼들의 위치가 변하지 않도록 설계.
   - 컬럼 순서 표준화: 현장 작업 흐름과 직관성을 고려하여 '완료' -> '공정' -> '보완' 순으로 배치. 엑셀 헤더, 메인 리스트, PDF 뷰어 하단 버튼 순서 모두 동일하게 유지.
   - 한글 입력 최적화: 비고(Remarks) 입력 시 한글 받침 씹힘 현상을 방지하기 위해 별도의 StatefulWidget(`_RemarksCell`)로 분리 구현.
   - 포커스 해제: 입력 완료(onSubmitted) 또는 화면 어딜터치 시 `_forgetFocus()`를 통해 키패드를 즉시 닫고 가상 포커스 노드(`_dummyFocusNode`)로 이동.
   - 정렬 로직: 모든 헤더 클릭 시 정렬 지원. 가상 번호(2-10 등)는 '-' 기준 숫자 비교(`_compareDisplayNo`) 로직 적용.
   - 품목코드 3단계 순환 정렬: 헤더 클릭 시 '오름차순 -> 내림차순 -> 정렬 해제(원래 순서)' 순으로 반복 순환하여 사용자 편의성 극대화.
   - 팝업 UI 개선: 공정, 보완, 완료 팝업 상단에 품목코드를 크게 표시하여 가독성 강화. 문구는 "입력시간 :"으로 통일.

4. 고급 필터 및 검색 시스템
   - 'No' 헤더 3단계 순환 필터:
     * 1단계: 전체 표시 (기본)
     * 2단계: 엑셀 원본 NO가 있는 항목만 표시 (헤더 노란색)
     * 3단계: '하위가 없는 원본' + '하위 항목들'만 표시 (헤더 텍스트 -No, 노란색)
   - 부분제목 다중 선택 필터링:
     * 부분제목 보기 모드에서 왼쪽 체크박스를 통해 여러 섹션을 동시에 선택 가능.
     * 선택 시 하단에 [선택 항목 보기], [모두보기], [모두 해제] 등 컨트롤 바가 나타남.
   - 헤더 리셋 기능: 완료, 공정, 보완 헤더 클릭 시 필터 팝업 하단에 [현재 리스트 항목 리셋] 버튼 추가. 현재 화면에 보이는 항목들만 대상으로 선택적 초기화 가능.
   - **선택 필터(Selection Filter)**: 선택 모드에서 항목 선택 후 [선택필터] 클릭 시, 해당 항목들이 포함된 **부분제목(섹션)과 함께** 리스트가 구성되며 선택 모드는 자동으로 종료됨.

5. 선택 모드 및 드래그 선택 (UX 고도화)
   - **선택모드**: 상단 "선택모드" 버튼을 통해 진입. 행의 어느 부위를 눌러도 체크가 토글됨.
   - **제스처 영역 분리**: 선택 모드에서 **'품목코드' 열은 수동 스크롤** 전용으로, **그 외 영역은 드래그 선택** 전용으로 분리하여 조작 충돌 방지.
   - **드래그 선택**: 행(품목코드 제외 영역)을 누른 채 아래로 드래그하면 항목들이 연속적으로 선택됨. 화면 끝에 닿으면 자동으로 스크롤이 내려가며 대량 선택 지원.
   - 상단 액션 바: 항목 선택 시 [선택필터](선택된 것만 보기)와 [삭제] 버튼 활성화.
   - 행 삭제 안내: 선택된 항목 삭제 시 하위 항목 포함 개수를 정확히 안내하며, 로컬 PDF 동기 삭제 옵션 제공.

6. PDF 뷰어 화면(PdfViewerScreen)
   - 헤더 정보 강화: 품목코드 옆에 (수량: X) 정보를 노란색(Colors.yellowAccent)으로 표시.
   - 이원화된 네비게이션: 돋보기 아이콘을 통한 '전체 품목 검색'과 좌측 하단 플로팅 화살표를 통한 '필터링된 순서 이동' 동시 지원.
   - 상태 동기화: 완료, 보완, 공정(커스텀 색상 반영), 비고 상태 즉시 메인과 연동. 완료 체크 시 보완 데이터만 초기화(공정 유지).

7. 공정 관리 및 색상 커스터마이징
   - 관리 로직: '완료' 항목은 항상 리스트 최하단에 고정되며 삭제가 불가능하도록 보호됨.
   - 색상 지정: 설정에서 15가지 색상 팔레트를 통해 공정별 고유 색상 지정 가능. 메인 리스트와 PDF 뷰어에 Accent Bar 형태로 반영.

8. QR/바코드 스캐너 및 자동 스크롤
   - 지능형 데이터 정제: 스캔된 코드에서 `<NUL>`, `<NULL>`, 제어 문자 등을 제거하고 접미사 `-S`를 자동 처리하여 순수 품목코드만 추출.
   - 즉시 이동 및 하이라이트: 스캔 성공 시 해당 항목으로 즉시 자동 스크롤하고 파란색 테두리로 강조 표시.
   - 줌(Zoom) 설정 연동: 스캐너 내에서 조절한 줌 배율이 설정 및 다음 스캔 시에도 유지되도록 SharedPreferences 연동.

9. 시간 기록 및 리셋 정책
   - 완료 시간 기록: 완료 체크 시 현재 시간을 `completeTime`에 기록. 완료 칸 롱프레스(1초) 시 시간 확인 팝업 출력.
   - 리셋 로직: '완료' 시 보완 데이터는 초기화하지만 공정 데이터는 유지하여 작업 이력 보존.
   - 시간 형식: 모든 시간은 `yyyy-MM-dd HH:mm` 형식으로 기록 및 저장.
   - **필터 리셋 시 선택 초기화**: '필터리셋' 버튼 클릭 시 모든 필터와 함께 선택된 항목들(`_selectedIndices`)도 초기화됨.

10. 파일 시스템 및 SMB 연동
   - 기본 경로: `/storage/emulated/0/Download/CheckSheet/`
   - SMB 브릿지: Kotlin 네이티브 핸들러를 통한 PC 공유폴더 탐색 및 양방향 파일 전송.
   - 엑셀 생성: 앱 내에서 직접 빈 양식의 엑셀 파일을 생성하여 즉시 작업 시작 가능.
