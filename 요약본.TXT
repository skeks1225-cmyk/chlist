📋 CheckSheet 앱 최종 기술 요약본 (Rebuild Guide)

1. 핵심 아키텍처 (Tech Stack)
* 프레임워크: Flutter (Dart) + Android Native (Kotlin)
* SMB 엔진: jCIFS-ng (v2.1.34) 단일 체제 (고성능/안정성)
  - 이유: Windows 10/11 및 Tailscale 환경에서 가장 안정적인 공유 목록 조회 및 파일 전송 지원.
* 데이터 처리: 
  - excel: 엑셀 읽기/쓰기 (TextCellValue 방식 사용)
  - flutter_pdfview: 네이티브 기반 PDF 뷰어 (자유로운 핀치 줌 및 자동 fit)
* 권한: MANAGE_EXTERNAL_STORAGE (안드로이드 11+ 직접 파일 제어)

2. 네이티브 엔진 설정 (The Heart: Kotlin)
* 무상태(Stateless) & 온데만드(On-Demand) 접속
  - 원칙: 모든 네트워크 작업 호출마다 IP/ID/PW를 전달하여 재접속 수행.
  - 안정성: ensureContext() 초기화는 반드시 백그라운드 쓰레드에서 실행 (NetworkOnMainThreadException 방지).
  - Throwable 캐칭: 모든 에러는 Throwable로 감싸서 Crash 방지.
* 지능형 파일 매칭: PC 파일명 대소문자 무시 (ignoreCase) 매칭 필수 적용.

3. 핵심 기능 로직 (Key Logic)
* 스마트 동기화 (Smart Sync)
  - 로컬과 PC 파일의 [용량 + 수정시간] 대조하여 바뀐 파일만 다운로드.
* PDF 전체 동기화 (Batch Processing)
  - 파일을 5개씩 묶어서 동시에 처리 (세미-병렬 방식)하여 5배 빠른 속도 제공.
* 내추럴 정렬 (Natural Sort)
  - 'No'와 '수량' 컬럼은 int.tryParse를 통한 숫자 크기 기반 정렬 적용.

4. UI/UX 디테일 (The Interface)
* SafeArea: 시스템 내비게이션 바와의 간섭 방지.
* 포커스 자동 해제: 
  - 비고란 외 다른 터치 발생 시, 가짜 포커스 존(Dummy FocusNode)으로 이동시켜 키패드 강제 해제.
* 다크모드 대응: 
  - Scaffold 배경색과 PDF 뷰어의 nightMode를 시스템 테마와 연동.

5. 빌드 설정 (The Shield: Gradle)
* Bouncy Castle 보안 충돌 해결
  - bcprov-jdk18on:1.78.1 강제 고정 (resolutionStrategy).
  - 주의: packaging 설정에서 인증서(*.RSA, *.SF)는 절대 삭제 금지.
* Proguard (R8) 설정: 
  - 최적화 비활성화(isMinifyEnabled = false)로 릴리즈 안정성 확보.

⚠️ 재구축 시 주의사항
1. jCIFS-ng 초기화는 반드시 suspend 함수 내부에서 진행할 것.
2. 모든 네트워크 작업은 반드시 Dispatchers.IO 문맥에서 실행할 것.
3. 빌드 시 라이브러리 간의 중복 클래스 충돌 방지를 위해 반드시 force 및 exclude 전략을 포함할 것.
